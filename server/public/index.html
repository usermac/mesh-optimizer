Quick Start: Missing Textures?

Why is my model white/grey?

If you uploaded an .obj file and the result has no color, you likely forgot the texture files!

The Rule: OBJ files need their associated material file (.mtl) and texture images (.jpg/.png) to display colors correctly.

How to fix it:

Option 1 (Recommended): Create a .zip file containing your model.obj, model.mtl, and all texture images. Drag the ZIP onto the upload area. This keeps everything organized together.

Option 2: Drag just the .obj file, then use the "Material (.mtl)" and "Textures (Max 50)" fields that appear below to add the additional files separately.

Option 3: Use a .glb file instead — it has textures built-in, so you only need one file.

Which Mode Should I Use?

Standard Decimation

Cost: 1 Credit

Best for: Reducing file size while keeping the exact same geometry structure. Good for CAD models or models that are already clean.

How it works: It simply removes polygons. It does not create new textures.

Premium Remesh (Bake)

Cost: 2 Credits

Best for: 3D Scans, Photogrammetry, and Organic shapes.

How it works: It builds a brand new "clean" mesh (Retopology) and "bakes" the details from the high-quality original onto the new one using Normal Maps. This is the professional workflow for games and AR.

Supported File Formats

  • GLB (Recommended): The best format. All geometry, materials, and textures are embedded in a single file. Just drag and drop.
  • GLTF: Supported, but note that GLTF files often reference external .bin buffer files. If your GLTF has external buffers, upload as a ZIP containing the .gltf and all .bin files together.
  • ZIP (Recommended for OBJ/GLTF): Zip your model file + all associated files (.mtl, .bin, textures) into one archive. Easiest way to keep everything organized together.
  • OBJ: Supported. After uploading, use the Material and Textures fields to add .mtl and .jpg/.png files separately (max 50 textures).
  • FBX: Supported, but textures may not transfer if they aren't embedded in the FBX.

Supported texture formats: .png, .jpg, .jpeg, .tga, .bmp, .tif, .tiff

API Documentation

📦 glTF 2.0 Compliant
🛡️ Files Auto-Delete After 1hr

Base URL: https://webdeliveryengine.com
Authentication: All protected endpoints require Authorization: Bearer YOUR_API_KEY

Workflow Overview

The API uses an asynchronous 3-step workflow:

  1. Submit: POST your model to /optimize and receive a job ID
  2. Poll: Check /job/:id until status is Completed or Failed
  3. Download: Fetch your optimized files from the URLs in the response (expires 1 hour after completion)

POST /optimize

Submit a 3D model for processing. Returns a job ID for polling. Requires authentication.

Request (multipart/form-data)

FieldTypeRequiredDescription
filefileYes3D model file (.obj, .fbx, .glb, .gltf), .zip, or multiple files (model + .mtl + textures). Max 50 texture files.
modestringNodecimate (default) or remesh
target_percentagefloatNoTarget quality as percentage (1-100) for decimate mode. E.g., 50 = keep 50% of faces.
target_facesintNoExact target face count for decimate mode. Alternative to target_percentage.
ratiofloatNo(Legacy) 0.0-1.0 quality ratio for decimate mode. Use target_percentage instead.
facesintNoTarget face count for remesh mode (default: 5000)
texture_sizeintNoTexture resolution for remesh: 1024, 2048, 4096 (default: 2048)
formatstringNoOutput format: glb, usdz, or both (default: glb)
callback_urlstringNoHTTPS URL to receive webhook on job completion/failure

Supported auxiliary files: .mtl, .png, .jpg, .jpeg, .tga, .bmp, .tif, .tiff, .bin

Quick Start Example

# Option A: Using a ZIP file (simplest approach)
curl -X POST 'https://webdeliveryengine.com/optimize' \
     -H 'Authorization: Bearer YOUR_API_KEY' \
     -F 'file=@model.zip' \
     -F 'mode=decimate' \
     -F 'target_percentage=50'

# Option B: Multi-file upload (OBJ with 6 textures)
curl -X POST 'https://webdeliveryengine.com/optimize' \
     -H 'Authorization: Bearer YOUR_API_KEY' \
     -F 'file=@model.obj' \
     -F 'file=@model.mtl' \
     -F 'file=@diffuse.png' \
     -F 'file=@normal.png' \
     -F 'file=@roughness.png' \
     -F 'file=@metallic.png' \
     -F 'file=@ao.png' \
     -F 'file=@emissive.png' \
     -F 'mode=decimate' \
     -F 'target_percentage=50'

# Option C: Using exact face count target
curl -X POST 'https://webdeliveryengine.com/optimize' \
     -H 'Authorization: Bearer YOUR_API_KEY' \
     -F 'file=@model.glb' \
     -F 'mode=decimate' \
     -F 'target_faces=25000'

# 2. Poll for completion (replace JOB_ID)
curl 'https://webdeliveryengine.com/job/JOB_ID'

# 3. Download results when complete
curl -O 'https://webdeliveryengine.com/download/JOB_ID/model_opt.glb'
curl -O 'https://webdeliveryengine.com/download/JOB_ID/model_opt.usdz'

Tip: ZIP files are often easier to manage and keep your textures organized together.
