Accessibility Statement

Last Updated: December 2025

♿ WCAG 2.1 Level AA Target

Our Commitment

MeshOpt is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone, applying the relevant accessibility standards, and ensuring our 3D optimization tools are usable by all.

We believe that technology should empower everyone, regardless of their abilities. Accessibility is not just a feature—it's a fundamental requirement.

Conformance Status

The Web Content Accessibility Guidelines (WCAG) defines requirements for designers and developers to improve accessibility for people with disabilities. It defines three levels of conformance: Level A, Level AA, and Level AAA.

MeshOpt aims to conform to WCAG 2.1 Level AA. This means we strive to meet all Level A and Level AA success criteria.

Measures We've Implemented

To ensure accessibility, we have taken the following measures:

Semantic HTML & Structure

  • Proper HTML5 semantic elements (<header>, <main>, <footer>, <aside>)
  • Logical heading hierarchy (h1 → h2 → h3)
  • Meaningful page titles and meta descriptions
  • Language attribute properly set (lang="en")

Keyboard Navigation

  • All interactive elements are keyboard accessible
  • Skip navigation link to jump to main content
  • Visible focus indicators on all focusable elements
  • Logical tab order throughout the application
  • Modal dialogs trap focus appropriately
  • Escape key closes modal dialogs

Screen Reader Support

  • ARIA landmarks for major page regions
  • ARIA labels on complex controls and interactive elements
  • Live regions (aria-live) announce dynamic content changes
  • Status messages for processing and file uploads
  • Proper labeling of all form inputs
  • Descriptive button and link text

Visual Design

  • Sufficient color contrast ratios (targeting WCAG AA standards)
  • Text remains readable when zoomed to 200%
  • Information not conveyed by color alone
  • Minimum touch target sizes (44x44 pixels) for mobile users
  • Responsive design works across devices and screen sizes

Forms & Interaction

  • Clear labels associated with all form fields
  • Required fields clearly indicated
  • Error messages are descriptive and helpful
  • Success confirmations announced to screen readers
  • Helper text provides additional context where needed

Testing & Validation

We test our application using:

  • Manual keyboard testing - Navigating the entire app without a mouse
  • Screen readers - NVDA (Windows), JAWS (Windows), VoiceOver (macOS/iOS)
  • Automated tools - axe DevTools, Lighthouse, WAVE
  • Code validation - W3C HTML and CSS validators
  • Browser testing - Chrome, Firefox, Safari, Edge (latest versions)

Known Limitations

Despite our best efforts to ensure full accessibility, some limitations may exist:

  • The 3D model viewer component relies on WebGL and may have limited accessibility for screen reader users. We provide text alternatives where possible.
  • Some third-party libraries (e.g., model-viewer) may have accessibility limitations outside our direct control.
  • Complex visual information (3D models) may be difficult to convey to users with visual impairments. We're exploring alternative representations.

We are actively working to address these limitations and welcome feedback on how we can improve.

Compatibility

MeshOpt is designed to be compatible with the following assistive technologies:

  • Recent versions of screen readers (JAWS, NVDA, VoiceOver, TalkBack)
  • Browser zoom and text resizing
  • Speech recognition software
  • Keyboard-only navigation

Supported browsers:

  • Chrome (latest 2 versions)
  • Firefox (latest 2 versions)
  • Safari (latest 2 versions)
  • Edge (latest 2 versions)

Ongoing Improvements

Accessibility is an ongoing journey, not a destination. We continuously monitor and improve our application based on:

  • User feedback and accessibility reports
  • Regular accessibility audits
  • Updates to WCAG guidelines and best practices
  • Advances in assistive technology
  • Community standards and recommendations

Formal Complaints

If you are not satisfied with our response to your accessibility feedback, you may escalate your concerns through our contact form, clearly marking your message as an "Accessibility Complaint." We take all such complaints seriously and will respond promptly.

Technical Specifications

Accessibility of MeshOpt relies on the following technologies to work with the particular combination of web browser and assistive technologies or plugins installed on your computer:

  • HTML5
  • CSS3
  • JavaScript (ES6+)
  • WAI-ARIA (Web Accessibility Initiative – Accessible Rich Internet Applications)

These technologies are relied upon for conformance with the accessibility standards used.
